Boinst's NAnt Extensions ======================== A collection of NAnt Extensions ------------------------------- [Download Source][3] - [Download Binary][2] - [View on GitHub][3] ## What is Boinst's NAnt TeamCity Extensions? This is a collection of extensions for NAnt, organised into the following assemblies. **Boinst.NAntExtensions** Handy extensions for NAnt. **Boinst.NAntExtensions.TeamCity** What you get here is a collection of NAnt Extensions that you can use when you're running an NAnt build under JetBrains' [TeamCity][1] CI Server. ## Installation Download the latest version from [our SourceForge site][2], or grab the [source from GitHub][3]. Extract the archive (or compile the extensions from source), and copy the .dll files into the `extensions\common\4.0` subdirectory of your NAnt installation. ## Boinst.NAntExtensions ### NAnt Mutex Task #### What is it good for? Run any block of actions within a global mutex. Particularly useful within a CI context, if you are manipulating some fixed location, or performing a system action like installing an MSI package. #### Invocation 

<mutex name="mymutexname" global="true" timeout="1000">
    ... do stuff here
</mutex>
* "name" is required. * "global" is "true" by default, and produces a global mutex. * "timeout" is "-1" (no time limit) by default, and is in milliseconds. ### NAnt Quiet Task #### What is it good for? Suppress the output of particularly verbose tasks. #### Invocation 

<quiet>
    ... do stuff here
</quiet>
## Boinst.NAntExtensions.TeamCity ### TeamCity Logger #### What is it good for? When you use TeamCity's inbuilt version of NAnt, you get your logs formatted nicely, particularly in the "Tree View" log panel. This logger allows you to get similar logging while allowing you the flexibility of using your preferred version of NAnt. #### Invocation Launch NAnt with the following syntax: `NAnt.exe -logger:Boinst.NAntExtensions.TeamCity.TeamCityLogger` #### See Also * [NAnt documentation on Loggers][4] ### Publish Artifacts to TeamCity #### What is it good for? This NAnt task allows you to publish artifacts to TeamCity while a build is in progress. #### Invocation #### See Also * [TeamCity documentation on publishing artifacts while a build is in progress][5] ## Problems?
